# Job Prompts

The user will be prompted with potential jobs they might wish to accept.

#### Requirements

* Glance-able (user can make a decision based off a glance)
* Quick to use
* Many prompts/realtime
* Few distractions (as user will likely use it whilst performing other jobs)

## Key Info

The key info a user will need to determine if they wish to take the job or not will be:

* Location (restaurant and customer)
* Capacity (amount of space the order will take)
* Fee
* Attributes of the order (hot, cold, delicate, etc.)
* Time (time required to complete this task)
* Order ETA (when the order should be ready to be picked up)
